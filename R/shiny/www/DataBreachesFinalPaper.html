Data Breaches

Motivation

Throughout the course, we have learned a lot from the data breaches data set.  We began with the goal of finding an intuitive way to show how prevalent data breaches are in today’s society.  Using statistical analysis and data visualization, we were able to learn a solid set of valuable information from the data.  We learned things about where breaches are occurring, about how breaches are on the rise, and about how breaches are on the rise.

Data Collection

We began our project by collecting data from the ID Theft Resource Center (ITRC).  The ITRC gathers data on identity theft, scams, fraud, and more.  The best part is that they make all the information publicly available.  The ITRC compiled data about data breaches that have occurred since 2005 in the form of PDF files, organized by year.  However, the links to the PDFs for 2005-2013 were broken.  This introduced our first issue.  To get around this difficulty, we implemented the Internet Wayback Machine, a data storage site that allows people to access previously posted data on the internet.  Thankfully, the ITRC website was cached and we were able to recover the missing PDFs.

Data Cleaning

Our next task was to scrape the data and import it into R.  Having the data in PDF format did introduce some difficulties during scraping due to the unreliable nature of .pdf to .txt conversion.  We used Zamzar.com to convert the data into text format at the cost of introducing some random character values outside of columns as well as other inaccuracies.  We had to clean this up manually.  Another method of .pdf conversion we used was “pdf2json.”  Using node.js, the data was matched up to predefined values and sorted into an object.

Results/Analysis[a]

Since we set out to look for trends that would increase the awareness of the prevalence of data theft, we performed numerous counts on the data.  This enabled us to see what kinds of companies were most susceptible to breaches, as well as where those companies were based.  We used this information to create numerous visualizations such as pie charts, bar graphs, and smooth line graphs.  We created a graph displaying the total number of breaches over time and in doing so found that there is a definite direct correlation.

Conclusion

Much of what little time we had remaining was spent perfecting our visualizations and presentation.  This necessitated getting the data into a dashboard as well as getting our data shiny interactive so that we could actively show people how things are changing over years and through different areas of business.  There were numerous decisions that had to go into the shiny implementations, such as which parts of our visualizations did we want to make interactive.  And then when committing different ideas to the visualizations, of course there were different challenges that came into play.
